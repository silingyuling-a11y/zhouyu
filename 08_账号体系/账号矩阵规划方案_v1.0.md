# 账号矩阵规划方案 v1.0

> 全网宽带获客项目 | 账号体系设计
> 原则：先精后多，先做深1-2个平台，再复制到其他平台

---

## 一、整体账号策略

### 1.1 账号分层

#### 主号（1个/平台）
- **定位**：官方形象、专业靠谱
- **内容**：干货科普、套餐介绍、客户案例
- **作用**：品牌建立、主要转化入口
- **要求**：实名认证、完善资料、定期更新

#### 副号（2-3个/平台）
- **定位**：真实用户、不同人设
- **内容**：个人体验、避坑分享、问答互动
- **作用**：辅助主号、制造真实感、搜索占位
- **要求**：不需要太专业，要像真实用户

#### 小号/N个（按需）
- **定位**：水军、提问者、互动者
- **内容**：提问、评论、顶贴、互动
- **作用**：提升互动率、制造热门假象、自问自答
- **要求**：账号要养，不要刚注册就用

## 三、账号养号策略

### 3.1 新号养号7天流程

| 天数 | 操作 | 注意事项 |
|------|------|----------|
| 第1天 | 注册账号，完善基本资料，随便刷刷内容 | 不要发任何广告、不要留联系方式 |
| 第2天 | 正常浏览，点赞收藏评论几个内容 | 模拟真实用户行为 |
| 第3天 | 关注几个同类型账号，继续互动 | 不要一下子关注太多 |
| 第4天 | 可以发第一条内容（纯干货/日常，不带广告） | 内容要正常，像普通人发的 |
| 第5天 | 发第二条内容，互动一下评论 | 慢慢增加发布频率 |
| 第6天 | 发第三条内容，可以稍微带点广告性质 | 不要太明显 |
| 第7天 | 正常发布，可以适当留联系方式 | 用暗语、引导私信 |

### 3.2 养号注意事项

1. **一机一号**：尽量一个手机登一个号，不要频繁切换
2. **固定IP**：不要频繁换IP、用VPN
3. **行为真实**：浏览、点赞、评论、关注都要像真实用户
4. **循序渐进**：广告不要加得太猛，慢慢加
5. **备份账号**：多准备几个备用号，以防被封

## 四、账号联动策略

### 4.1 内容联动
- 主号发干货内容
- 副号在评论区提问、互动
- 小号附和、顶贴、制造热门感

### 4.2 引流联动
- 平台内：副号引导到主号
- 跨平台：知乎/小红书引导到抖音看视频
- 最终都引导到微信私域

### 4.3 风险防控
- 主号要稳，不要做太冒险的操作
- 副号可以稍微激进一点测试
- 小号用来做风险高的操作
- 永远有备用账号可以顶上
